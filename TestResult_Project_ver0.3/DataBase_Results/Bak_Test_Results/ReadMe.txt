说明:

备份策略:将当前的测试结果收集后，立即进行分类备份，
         为方便查询，分为2种情况:
         Base:含有基本的测试用例名称及结果信息
         Detail:含有较详细的测试用例信息及结果信息

一、结果表

   分了两种情况，一个是基本表，一个是详细表
   暂命名为:Bak_版本号_Base_ALL
            Bak_版本号_Detail_ALL

   其中,版本号现在分两种情况:
   a)OS
     测试类型为操作系统，则数据表名中含有的是OS版本号

   b)Kernel
     测试类型为内核，则数据表名中含有的是内核版本号

   举例:
         Bak_20200103-RC1_Base               //备份的基本信息表,OS版本号:20200103-RC1
         Bak_3.10.84-23.fc21.loongson.1_Base //备份的基本信息表，内核版本号:3.10.84-23.fc21.loongson.1
